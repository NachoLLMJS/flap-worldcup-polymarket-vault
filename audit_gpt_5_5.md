#### H-03: Auxiliary betting vault cancellation and outcome arrays remain product-level audit risks

**Severity**: High  
**Status**: Open for product audit; non-applicable to Flap V2 if kept auxiliary  
**File**: `foundry/worldcup-betting/src/WorldCupBettingVault.sol:103-138`, `289-312`, `315-326`  
**Rule**: 003 fairness / general Solidity DoS

**Description**

`WorldCupBettingVault` is intentionally auxiliary and not the Flap-submitted tax vault. Still, for the whole product:

- `cancelMarket()` lets operator/Guardian cancel any unresolved market.
- `placeBet()` immediately sends a 1% fee to the hardcoded fee recipient.
- `refund()` returns only net stake, so cancelled users do not recover the fee.
- `createMarket()` accepts unbounded `outcomeTeamIds`; later `refundable()` and `getMarket()` loop over all outcomes.

**Impact**

A privileged operator can cancel after collecting fees, creating a fairness issue. An excessive outcome list can also make refund/view functions too gas-heavy.

**Recommendation**

For product audit readiness, either:

- only allow cancellation before bets exist;
- escrow fees until normal resolution and refund gross deposits on cancellation;
- restrict cancellation to documented oracle-failure states and/or Guardian-only delayed path;
- add a hard `MAX_OUTCOMES` cap.

### Medium Issues

#### M-01: `time` schema fields map to `uint64` ABI outputs

**Severity**: Medium  
**Status**: Open  
**File**: `contracts/WorldCupPolymarketVault.sol:45-47`, `270-274`, `374-384`  
**Rule**: UI schema compatibility

**Description**

The schema uses `fieldType = "time"` for:

- `matchStartTime`
- `bettingCloseTime`
- `settlementEarliestTime`

But the Solidity ABI returns `uint64` for `getMarketTiming()` and inside the `MarketMapping` struct returned by `getMarket()` / `getMarkets()`.

The Flap UI docs describe `time` as a UI alias for `uint256`. The skill notes warn that strict generic UI tooling may expect `uint256` for `time`.

**Impact**

The UI may render fine if it tolerates smaller integer widths in outputs, but this is not a strict best-practice pass.

**Recommendation**

Change UI-facing wrapper return types to `uint256` while keeping internal storage as `uint64`, e.g.:

```solidity
function getMarketTiming(uint256 index)
    external
    view
    returns (uint256 matchStartTime, uint256 bettingCloseTime, uint256 settlementEarliestTime)
{
    require(index < markets.length, "market missing");
    MarketMapping storage item = markets[index];
    return (uint256(item.matchStartTime), uint256(item.bettingCloseTime), uint256(item.settlementEarliestTime));
}
```

For `getMarket()`/`getMarkets()`, either expose a UI-specific struct with `uint256` time fields or change the schema type from `time` to a raw integer type if Flap confirms `uint64` output support.

## Centralization Analysis

### Admin Controls

- `operator` or Guardian can call `setOperator`, `setBettingVault`, `forwardTaxRewardsToBetting`, `upsertMarket`, and `refreshSettlement`.
- Guardian is hardcoded through Flap base contract per chain.
- Emergency withdrawals are Guardian-only.

### Upgrade Mechanisms

- The Flap vault uses EIP-1167 clones pointing at an immutable implementation address stored in the factory.
- EIP-1167 clones are not considered upgradeable for Rule 009.
- Existing clones cannot be changed if the implementation has a bug; relaunch/factory update is required.

### Emergency Functions

- Native and token emergency recovery functions exist and are Guardian-only.
- They drain the full balance and use `nonReentrant`.

### Decentralization Recommendations

- Disclose operator powers.
- If low-risk badge is desired, ask Flap whether operator metadata/reward-forwarding controls need additional constraints or launch-plan commitments.

## Gas Optimization Recommendations

- `receive()` is already minimal.
- `description()` dynamically converts balance to string; this is view-only and not a hot path.
- `getMarkets(offset, limit)` caps `limit <= 100`, which is good for view pagination.
- If converting UI time wrappers to `uint256`, keep internal storage as `uint64` for packing/storage efficiency.
